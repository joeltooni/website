## CLASS PHASE

### Opening & Settling In
**Time: 1:30 PM – 1:40 PM** (10 minutes)

- Students arrive & settle in
- Icebreaker conversation: Ask about their weekend plans, get to know them

---

### Coding Problem-Solving & Design (Main Activity)
**Time: 1:40 PM – 2:20 PM** (40 minutes)

#### Content & Design Overview
**Time: 1:40 PM – 1:55 PM** (15 minutes)

- Top-down design overview (how to break programs into smaller parts)
- Guiding questions for each design level provided on slide (feel free to add your own)
- **Key emphasis:** Problem-solving and creativity happen here – it's hard, and that's OK

#### Spotlight Activity: Annie Easley & Space Game
**Time: 1:55 PM – 2:05 PM** (10 minutes)

- Introduce practice activity featuring Annie Easley (NASA CS – space-themed game)
- Play/demo the game so students understand what they're pseudocoding
- *Note: Video rules differ slightly from implementation students will code*
- Review coding instructions in handout with students

#### Hands-On Activity: Breaking Down the Program
**Time: 2:05 PM – 2:20 PM** (15 minutes)

- Students work through breaking down the program with guidance
- TAs actively jump in and help – encourage students to take initiative and ask how things are going
- Students can test ideas in Scratch but should NOT code up the entire game unless on pace to finish
- **Key Focus:** Code structure and planning

---

### Introduction to Python
**Time: 2:20 PM – 3:00 PM** (40 minutes)

**Lecture & Connections:**
- Connect to prior learning: GH made compilers, which enabled high-level languages like Python
- Lecture content is in slides
- Incorporate live coding demos where possible

**Distribution:**
- Pass out Syntax Notesheet

**Interactive Learning:**
- Students work through Syntax Notesheet as you talk
- Use lecture to reiterate building blocks and introduce Python syntax
- Reference back to Scratch blocks to contextualize syntax

**Type-Along Exercises:**
- Type sort code snippets into Trinket
- Ask students to trace the code
- Run code to show output
- Each snippet demonstrates what changing one parameter does
- See documentation for code snippets
- Goal: Get students comfortable typing out Python code

**If Extra Time:**
- Have simple coding exercises available for them to try
